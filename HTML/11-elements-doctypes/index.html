DOCTYPE and HTML Elements

1️⃣ DOCTYPE Declaration

The DOCTYPE for HTML5 is:

<!DOCTYPE html>
Why use DOCTYPE?
  • Tells browser to use standards mode
  • Ensures consistent rendering across browsers
  • Prevents quirks mode (old browser behavior)
  • Required for valid HTML5

2️⃣ Block-level Elements

Block elements start on a new line and take full width.

<div> <p> <h1>–<h6> <section> <article> <header> <footer> <main> <ul> <ol>

Example:

I am a BLOCK element (DIV) - I take full width!
I am another BLOCK element - I start on a new line!

3️⃣ Inline Elements

Inline elements do NOT start on a new line. They only take the space they need.

<span> <a> <strong> <em> <img> <code> <small> <b> <i>

Example:

This is an inline SPAN and this is a STRONG tag and here's a LINK - all in the same line!

4️⃣ Self-closing (Void) Elements

These elements have NO closing tag. They are self-contained.

<br> <hr> <img> <input> <meta> <link> <area> <col>

Examples:

Line 2 (used <br> to break line)
Horizontal line above (used <hr>)

5️⃣ Semantic vs Non-semantic Elements

Semantic Elements

Describe meaning and purpose

<header> <nav> <main> <section> <article> <aside> <footer> <figure>
Benefits:
  • Better SEO
  • Accessibility
  • Code readability

Non-semantic Elements

No specific meaning - just containers

<div> <span>
Use when:
  • No semantic meaning
  • Just for styling
  • Grouping elements

Semantic HTML Example:

<header>
  <nav>Navigation</nav>
</header>

<main>
  <article>
    <section>Content</section>
  </article>
</main>

<footer>Footer</footer>
